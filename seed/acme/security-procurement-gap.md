# Security and Procurement Gap

Prepared by: Priya Raman
Date: November 12, 2025

## Problem

Acme has no single owner for late-stage security and procurement work. Requests bounce between customer success, finance, and whoever happens to be online when a customer asks for a security packet, vendor form, or legal attachment.

## What this looks like in practice

- Marcus often opens the thread because the issue starts with a customer.
- Priya gets pulled in because support has the cleanest incident history.
- Femi gets tagged when the request smells commercial, even if it started as a security questionnaire.
- Jordan gets pulled in too early when teams are unsure who can actually approve an exception.

## Why it matters

The same ownership gap shows up in vendor work. The Dataflow renewal was treated as procurement by one team and as security diligence by another, which delayed negotiation prep and left nobody clearly accountable for the final package.

## Recommendation

- Create one named owner for security questionnaires and procurement handoffs.
- Separate incident-response facts from pricing-exception approvals in the escalation template.
- Retire the onboarding shortcut that implies executive involvement can happen before finance review.
